<a id="프로젝트-개요"></a>
<details open>
<summary><b>📌 프로젝트 개요</b></summary>
<br>

### 프로젝트 배경
여러 OTT, 음악, AI, 클라우드 구독 서비스가 늘어나면서 사용자는 각 플랫폼에 흩어진 결제 정보와 이용 현황을 따로 관리해야 하는 불편을 겪고 있습니다.  
Subees 프론트엔드는 이러한 문제를 해결하기 위해 구독 등록, 결제 캘린더, 소비 분석, 알림, 커뮤니티, AI 추천 기능을 하나의 사용자 경험으로 연결하는 것을 목표로 합니다.

### 프론트엔드 역할
- 사용자 인증 흐름과 세션 상태를 관리합니다.
- 구독/결제/커뮤니티/추천 데이터를 화면에 맞게 가공해 제공합니다.
- 대시보드, 캘린더, 알림 센터 등 핵심 화면을 반응형 UI로 구성합니다.
- 백엔드 API와 연동하여 실제 데이터를 기반으로 동작하는 화면을 제공합니다.

### 현재 구현 범위
- 홈 화면 프리뷰
- 로그인 / 회원가입
- 대시보드
- 구독 목록 / 구독 추가
- 결제 캘린더
- 알림 센터
- 커뮤니티 / 스크랩
- AI 추천 / 추천 기록 / 추천 상세
- 결제 카드 관리
- 마이페이지 / FAQ

</details>

---

<a id="서비스-목표"></a>
<details>
<summary><b>🎯 서비스 목표</b></summary>
<br>

- 사용자가 여러 구독 서비스를 한 화면에서 쉽게 조회하고 관리할 수 있는 UI를 제공합니다.
- 결제 예정일, 카테고리별 소비, 월별 추이 등을 직관적으로 확인할 수 있도록 시각화합니다.
- 인증 상태, 입력 검증, 예외 처리까지 포함한 안정적인 사용자 흐름을 제공합니다.
- 백엔드 기능을 실제 서비스 경험으로 연결하는 프론트엔드 구조를 구축합니다.

</details>

---

<a id="사용자-흐름"></a>
<details>
<summary><b>📖 사용자 흐름</b></summary>
<br>

1. 사용자는 홈 화면에서 서비스 소개와 주요 기능 프리뷰를 확인합니다.
2. 비회원은 회원가입 후 로그인하고, 회원은 바로 로그인합니다.
3. 로그인 이후 대시보드에서 월별 지출 현황과 결제 예정 구독을 확인합니다.
4. 사용자는 구독 목록에서 구독을 조회하고, 새 구독을 추가합니다.
5. 결제 캘린더에서 날짜별 결제 예정 내역과 소비 흐름을 확인합니다.
6. 알림 센터에서 당일 또는 결제일 기준 사전 알림 내역을 확인합니다.
7. 커뮤니티에서 게시글을 작성, 조회, 수정, 삭제하거나 스크랩합니다.
8. AI 추천 화면에서 조건 기반 추천을 생성하고 결과를 저장·조회합니다.
9. 마이페이지에서 회원 정보와 결제 카드 정보를 관리합니다.

</details>

<a id="주요-기능"></a>
<details>
<summary><b>🧩 주요 기능</b></summary>
<br>

### 1. 인증 및 세션 관리
- 로그인 / 회원가입 화면 제공
- JWT 기반 로그인 세션 저장
- 토큰 만료 시 세션 정리 후 로그인 화면으로 이동
- 비로그인 사용자의 접근 제한 라우팅 처리

### 2. 대시보드
- 월별 소비 요약
- 카테고리별 소비 분석
- 결제 예정 구독 목록
- 인사이트 패널 및 홈 프리뷰 연계

### 3. 구독 관리
- 구독 목록 조회
- 구독 상세 패널 표시
- 카테고리 / 결제 주기 / 금액 입력 기반 구독 추가
- 서비스 이미지 자동 매핑

### 4. 결제 캘린더 및 알림
- 달력 기반 결제 일정 확인
- 선택 날짜 상세 내역 패널
- 소비 추이 패널
- 알림 센터 및 헤더 알림 팝오버 제공

### 5. 커뮤니티 및 AI 추천
- 게시글 CRUD 및 스크랩 기능
- 추천 조건 입력, 결과 조회, 저장, 기록/상세 조회
- FAQ 페이지 제공

</details>

<a id="실행-방법"></a>
<details>
<summary><b>🚀 실행 방법</b></summary>
<br>

### 1. 실행 환경
- Node.js 18 이상
- npm
- 백엔드 서버 실행 상태 필요

### 2. 설치
```bash
npm install
```

### 3. 환경 변수 설정
프로젝트 루트에 `.env` 파일을 생성합니다.

```env
VITE_API_BASE_URL=http://localhost:8080
```

### 4. 개발 서버 실행
```bash
npm run dev
```

### 5. 배포용 빌드
```bash
npm run build
```

### 6. 미리보기 실행
```bash
npm run preview
```

</details>

<a id="api-연동-및-환경-변수"></a>
<details>
<summary><b>🔗 API 연동 및 환경 변수</b></summary>
<br>

### API 기본 주소
Axios 인스턴스는 아래 환경 변수를 기준으로 백엔드와 통신합니다.

```env
VITE_API_BASE_URL=http://localhost:8080
```

### 주요 연동 도메인
- 인증: 로그인, 로그아웃, 회원가입, 중복 확인
- 대시보드: 월별 소비 요약, 인사이트, 결제 예정 목록
- 구독: 목록 조회, 등록, 상세 조회
- 캘린더: 날짜별 결제 일정, 소비 추이
- 알림: 알림 목록, 상태 반영
- 커뮤니티: 게시글 / 스크랩
- 추천: AI 추천 생성, 저장, 기록, 상세
- 마이페이지 / 카드: 회원 정보, 결제 카드 관리

### 세션 처리 방식
- 로그인 성공 시 세션 정보를 로컬 스토리지에 저장합니다.
- 요청 시 Authorization 헤더를 자동으로 주입합니다.
- 만료된 토큰 응답이 오면 세션을 정리하고 로그인 화면으로 이동합니다.

</details>
