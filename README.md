<a id="프로젝트-개요"></a>
<details open>
<summary><b>📌 프로젝트 개요</b></summary>
<br>

- **서비스 명칭:** Subees
- **서비스 소개:** 흩어져 있는 구독 정보를 한곳에서 통합 관리하고 시각화된 대시보드를 제공하는 프론트엔드 시스템입니다.
- **주요 가치:** - **시각화:** 차트와 대시보드를 통한 지출 직관화
  - **반응성:** 다양한 디바이스 환경에 대응하는 UI 제공
  - **사용자 경험:** 간편한 구독 등록 및 알림 인터페이스 구축

</details>

---

<a id="서비스-목표-및-요구사항"></a>
<details>
<summary><b>🎯 서비스 목표 및 요구사항</b></summary>
<br>

- 사용자가 편리하게 구독 정보를 입력하고 관리할 수 있는 인터랙티브한 UI를 구축합니다.
- 실시간 유효성 검사 및 에러 핸들링을 통해 안정적인 사용자 입력을 보장합니다.
- 복잡한 데이터(결제일, 금액)를 캘린더 및 통계 그래프로 변환하여 시각적 만족도를 높입니다.
- [요구사항 명세서 바로가기](https://docs.google.com/spreadsheets/d/1t28YAF3teou6grdUzRbnRs2NyKi5boFY/edit?usp=sharing)

</details>

<summary><b>🎨 화면 및 기능 설계서 (필수)</b></summary>
<br>

- [Figma 화면설계서 바로가기](https://www.figma.com/design/8EOuGl8Jxtz5RJapDro4ky/Subees-화면-및-기능설계서)

**주요 기능 화면**
1. **대시보드:** 월별 소비 차트 및 카테고리 분석
2. **구독 목록:** 카드형/리스트형 구독 현황 조회 및 CRUD
3. **구독 캘린더:** 날짜별 결제 예정 내역 확인
4. **마이페이지:** 프로필 및 알림 설정 관리

</details>

<a id="api-명세서"></a>
<details>
<summary><b>📑 API 명세서 및 연동 가이드</b></summary>
<br>

- [API 명세서 바로가기 (Notion)](https://www.notion.so/api-31e712dca4bf807e97fbe583efc7e99e)

**연동 전략**
- **인증:** JWT 토큰 기반의 Private Router 및 Axios Interceptor 구현
- **데이터 가공:** API 응답 데이터를 UI 컴포넌트 규격에 맞춰 가공하는 커스텀 훅(Hook) 레이어 활용
- **에러 핸들링:** Global Error Boundary 및 Toast 메시지 알림 처리

</details>

<a id="데이터-구조-정의"></a>
<details>
<summary><b>🗺️ 데이터 구조 정의 (ERD 참고)</b></summary>
<br>

- [ERD 바로가기](https://www.erdcloud.com/d/osSpqKzTmS8zueTJs)

프론트엔드에서는 백엔드 ERD를 기반으로 데이터 타입을 엄격하게 관리합니다.

| 도메인 | 주요 Interface / Type |
| :--- | :--- |
| **User** | id, nickname, email, profileImg |
| **Subscription** | subId, itemName, price, cycle, nextDate |
| **Community** | postId, title, content, writer, scrapCount |

</details>

<a id="테스트-계획-및-결과 보고서"></a>
<details>
<summary><b>🧪 테스트 계획 및 결과 보고서</b></summary>
<br>

- **단위 테스트:** 주요 비즈니스 로직(날짜 계산, 금액 합산) 검증 (Jest)
- **컴포넌트 테스트:** 공통 UI 컴포넌트 및 유저 인터랙션 검증 (RTL)
- **E2E 테스트:** 회원가입부터 구독 등록까지의 핵심 유저 시나리오 검증 (Cypress/Playwright)
- **상태:** ✅ UI 렌더링 검증 완료 / API 통합 테스트 진행 중

</details>

<a id="프로젝트-마무리-회고"></a>
<details>
<summary><b>📝 프로젝트 마무리 회고 및 향후 확장 계획</b></summary>
<br>

- **회고:** 사용자 피드백을 반영한 모바일 최적화 및 접근성 개선 성과
- **확장 계획:** - 다크모드/라이트모드 테마 고도화
  - PWA 도입을 통한 네이티브 앱 수준의 푸시 알림 강화
  - 차트 라이브러리(Recharts)를 활용한 분석 데이터 시각화 강화

</details>
